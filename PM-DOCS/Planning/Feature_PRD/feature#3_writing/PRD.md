# F3. 글쓰기 & 발행 PRD

## 1. 개요

- **Feature명**: 글쓰기 & 발행
- **소속 Milestone**: M-001 (MVP 핵심 기능)
- **한 줄 요약**: 작가가 글쓰기에 몰입한 채 유료/무료 발행과 시리즈 관리를 자연스럽게 완료하는 에디터
- **라벨**: `feature:writing`

### 배경 및 목적

Readly의 콘텐츠 생산 파이프라인 핵심. CIS Design Thinking에서 도출한 "방해하지 않는 도구" + "독자 관점의 결과물을 만드는 도구"라는 에디터 핵심 가치를 구현한다. 작가의 글 생산량은 에디터 UX에 직결되며, 발행 과정의 복잡성이 콘텐츠 양을 좌우한다는 핵심 인사이트를 기반으로 설계한다.

### Design Principles (CIS 기반)

| #   | 원칙                                      | 설명                                                           |
| --- | ----------------------------------------- | -------------------------------------------------------------- |
| 1   | 흐름 우선 (Flow First)                    | 글쓰기 몰입을 방해하는 모든 요소를 제거하거나 자동화           |
| 2   | 의도 추론 (Intent Inference)              | 게시선 삽입 = 유료, 삭제 = 무료. 작가의 행동에서 의도를 읽는다 |
| 3   | 독자 거울 (Reader Mirror)                 | 미리보기로 독자에게 어떻게 보이는지 즉시 확인                  |
| 4   | 보이지 않는 안전망 (Invisible Safety Net) | 다층 자동 저장으로 글 유실을 원천 차단                         |
| 5   | 콘텐츠 격상 (Content Elevation)           | Smart Default로 메타데이터 부담을 최소화, 콘텐츠 품질에 집중   |

### 의존성

- **선행**: F1 인증 & 계정 관리 (approved), F2 서점 시스템 (decomposed) -- bookstoreId 필수
- **후행**: F4 시리즈 관리

### F2 선행 연동 필요 항목

F3 구현에 착수하기 전 F2에서 반드시 제공해야 할 항목:

| #    | 항목                    | F2 현재 상태 | 필요 조치                                    |
| ---- | ----------------------- | ------------ | -------------------------------------------- |
| F2-1 | BookstoreEntity 존재    | PRD 확정     | F3 시작 전 bookstores 테이블 생성 완료 필요  |
| F2-2 | PostEntity.bookstoreId  | PRD 확정     | posts 테이블에 bookstoreId FK 추가 완료 필요 |
| F2-3 | 서점 보유 여부 확인 API | PRD 확정     | bookstore.hasBookstore 프로시저 제공 필요    |

### MVP 제약 (CIS 기반)

- 에디터 엔진은 CKEditor 5 기반 WYSIWYG (기존 기술 선택 유지)
- 유료/무료 전환은 게시선(paywall line)으로만 결정 (별도 토글 없음)
- 발행 디폴트는 무료 + 전체공개 + 단편
- 전자책 뷰어 자체는 Out of Scope (안내 배지만 표시)
- 이미지 고급 편집 Out of Scope (삽입만 가능)
- 유료 -> 무료 전환은 작가 재량 (기구매자 존재 시 경고 팝업)

## 2. 목표 & 성공 지표

### 비즈니스 목표

작가의 글쓰기 -> 발행 플로우를 최소 마찰로 완료하게 하여 콘텐츠 생산량을 극대화한다. "글만 쓰면 발행은 자동"이라는 경험을 통해 첫 글 발행까지의 허들을 최소화한다.

## 4. 유저 플로우

### 메인 플로우

**Flow 1: 새 글 작성 (US-1, US-6) [MVP]**

```
[서점 관리/마이페이지] -> [글쓰기 버튼 클릭] -> [에디터 진입]
  -> [임시저장 있음?]
    -> YES -> 토스트: "이전 글이 있습니다. 이어서 쓸까요?"
      -> [불러오기] -> 에디터에 내용 복원
      -> [새 글] -> 빈 에디터
    -> NO -> [빈 에디터]
  -> [제목 + 본문 작성] -> [30초마다 자동 저장 (백그라운드)]
```

**Flow 2: 유료 게시선 설정 (US-2) [MVP]**

```
[에디터 툴바] -> [게시선 삽입 클릭]
  -> [커서 위치에 유료 게시선 삽입]
  -> 상태 바: "유료 발행 예정"
  -> [게시선 드래그로 위치 조정] 또는 [게시선 삭제]
    -> 삭제 시: 상태 바 정상 복귀 (무료)
```

**Flow 3: 발행 (US-3, US-4, US-7, US-8, US-9) [MVP]**

```
[에디터] -> [발행하기 클릭] -> [발행 시트(바텀시트) 열림]
  -> 분류: [단편(기본) / 시리즈 선택]
    -> 시리즈 선택 시: [기존 시리즈 드롭다운]
    -> [+ 새 시리즈 만들기] -> [시리즈명 입력 -> 즉석 생성] -> 회차 #1 자동 부여
    -> 기존 시리즈 선택 시: 다음 회차 번호 자동 부여
  -> 가격: 게시선 있으면 "유료" 자동 + 가격 입력 / 없으면 "무료" 자동
  -> 접근 권한: [전체공개(기본) / 구독자 전용 / 구매자 전용 / 비공개]
  -> 메타데이터:
    -> 썸네일: [자동 생성 or 직접 업로드]
    -> 요약문: [본문 첫 200자 자동 추출, 수정 가능]
    -> 태그: [태그 입력]
  -> 전자책 뷰어 안내 배지 (조건 충족 시 표시)
  -> [미리보기] -> Flow 4
  -> [발행하기] -> [발행 완료] -> [임시저장 자동 정리]
```

**Flow 4: 미리보기 (US-5) [MVP]**

```
[발행 시트] -> [미리보기 클릭]
  -> [미리보기 화면]
    -> 독자 뷰: 제목 + 작가명 + 태그 + 본문
    -> 게시선 있는 경우: 무료 영역 + 게시선 + 유료 영역 블러 처리 + "구매하면 볼 수 있어요" 안내 (AC-5.2 참조)
  -> [에디터로 돌아가기]
```

> **"작가 뷰" 탭 미포함 (의도적 제외):** MVP에서는 독자 뷰 미리보기만 제공한다. "작가 뷰"는 실질적으로 에디터 화면 자체이므로 별도 탭이 불필요하다. 작가 전용 미리보기 탭(통계, 수익 예측 등)은 Growth 단계에서 검토한다.

**Flow 5: 임시저장 관리 (US-6) [MVP]**

```
[에디터] -> [임시저장 버튼 클릭 또는 Cmd/Ctrl+S]
  -> [수동 임시저장 실행] -> 토스트: "임시저장 완료"
[에디터] -> [임시저장 목록 버튼 클릭]
  -> [임시저장 모달]
    -> 목록: 최근 5개 (제목, 저장 시간, 글자 수, 자동/수동 구분)
    -> [불러오기] -> "현재 내용을 대체합니다" 확인 -> 에디터에 로드
    -> [삭제] -> 목록에서 제거
```

**임시저장 5개 제한 기준:**

- 새 글 (postId=null): 사용자별 최대 5개. 새 글 임시저장이 5개를 초과하면 가장 오래된 자동 저장본부터 삭제.
- 수정 글 (postId=특정값): 글별 최대 5개. 특정 글의 수정 임시저장이 5개를 초과하면 가장 오래된 자동 저장본부터 삭제.
- 총 임시저장 수 = 새 글 임시저장(최대 5개) + 각 수정 글별 임시저장(각 최대 5개). 전체 상한은 별도로 두지 않는다.

**Flow 6: 기존 글 수정 (US-10) [MVP]**

```
[서점 관리 > 작품 목록] -> [작품 선택 > 수정]
  -> [에디터에 기존 내용 로드]
  -> [현재 발행본 스냅샷 자동 생성 (원본 보존)]
  -> [수정 작업] -> [30초 자동 저장은 수정본에 대해 동작]
  -> [발행 시트에서 업데이트]
  -> [수정 취소 시: 원본 스냅샷으로 복원 가능]
```

**수정 시 임시저장 전략:**

- 수정 시작 시: 현재 발행본의 스냅샷을 자동 생성하여 원본을 보존한다 (type: 'edit_snapshot').
- 수정 중: 30초 자동 저장은 수정본에 대해 동작하며, 해당 postId에 연결된 임시저장으로 관리된다.
- 수정 취소: 원본 스냅샷(edit_snapshot)을 불러와 에디터에 로드하여 변경 이전 상태로 복원할 수 있다.
- 수정 발행 완료: 해당 postId의 모든 임시저장(스냅샷 포함)이 자동 정리된다.

**Flow 7: 유료 -> 무료 전환 경고 [MVP]**

```
[유료 발행 글(기구매자 존재)] -> [에디터에서 게시선 삭제 시도]
  -> [경고 팝업: "nn명이 이미 구매했습니다. 무료로 전환하시겠습니까?"]
  -> [확인] -> 게시선 삭제, 무료 전환
  -> [취소] -> 게시선 유지
```

### 대안 플로우

- 자동 저장 실패 시: 재시도 3회 -> 실패 시 "저장 실패" 토스트 + 수동 저장 유도
- 발행 시 제목 미입력: 발행 버튼 비활성화 + "제목을 입력해주세요" 안내
- 시리즈 회차 충돌: 서버 측 자동 증분으로 충돌 방지
- 대용량 본문 (>100KB): 에디터 성능 경고 + 자동저장 간격 60초로 조정
- CKEditor 로드 실패: 재로드 안내 버튼 표시
- 서점 미보유 유저가 /write 접근: "서점을 먼저 오픈해주세요" 안내 + 서점 오픈 페이지로 이동

## 7. 수용 기준 (Acceptance Criteria)

### US-1: 글쓰기 에디터

- **AC-1.1**: Given 서점을 보유한 작가가 로그인 상태일 때, When "글쓰기" 버튼을 클릭하면, Then CKEditor 기반 에디터가 로드되고 제목+본문을 입력할 수 있다.
- **AC-1.2**: Given 에디터가 로드되었을 때, When 기본 서식(B/I/U, 제목, 링크, 이미지 삽입)을 적용하면, Then WYSIWYG으로 즉시 반영된다.
- **AC-1.3**: Given 서점을 보유하지 않은 유저가 /write에 접근할 때, When 페이지가 로드되면, Then "서점을 먼저 오픈해주세요" 안내와 함께 서점 오픈 페이지로 이동한다.

### US-2: 유료 게시선

- **AC-2.1**: Given 에디터에서 글을 작성 중일 때, When 툴바의 "게시선" 버튼을 클릭하면, Then 커서 위치에 유료 게시선이 삽입되고 상태 바에 "유료 발행 예정"이 표시된다.
- **AC-2.2**: Given 게시선이 삽입된 상태에서, When 게시선을 드래그하면, Then 게시선 위치가 본문 내에서 변경된다.
- **AC-2.3**: Given 게시선이 삽입된 상태에서, When 게시선을 삭제하면, Then 상태 바가 정상 복귀(무료)하고 발행 시트의 가격도 무료로 전환된다.
- **AC-2.4**: Given 유료 발행된 글에 기구매자가 존재할 때, When 게시선을 삭제하려 하면, Then "nn명이 이미 구매했습니다" 경고 팝업이 표시되고 확인/취소를 선택할 수 있다.

### US-3: 발행

- **AC-3.1**: Given 작가가 글 작성을 완료하고 "발행하기"를 클릭할 때, When 발행 시트가 열리면, Then 분류/가격/접근권한/메타데이터 설정이 표시된다.
- **AC-3.2**: Given 게시선이 있는 글의 발행 시트에서, When 가격 영역을 확인하면, Then "유료"가 자동 선택되어 있고 가격 입력이 가능하다.
- **AC-3.2.1**: Given 유료 가격을 입력할 때, When 가격을 설정하면, Then 다음 정책이 적용된다:
  - 최소 가격: 100원
  - 최대 가격: 100,000원
  - 가격 단위: 100원 단위 (예: 100, 200, 300, ..., 100000)
  - 기본 가격 (Smart Default): 유료 게시선 삽입 시 300원이 자동 입력됨
  - 통화: KRW (원)
  - F2 서점 기본 가격 연동: 서점에 PublishDefaultEntity.defaultPrice가 설정되어 있으면 해당 값을 기본 가격으로 사용 (300원 대신)
  - 정책 위반 입력 시: 발행 버튼 비활성화 + "100원~100,000원 사이, 100원 단위로 입력해주세요" 안내
  - 무료 글(게시선 없음)은 price 필드를 전송하지 않으며, 기존 default 0이 유지됨
- **AC-3.3**: Given 게시선이 없는 글의 발행 시트에서, When 가격 영역을 확인하면, Then "무료"가 자동 선택되어 있다.
- **AC-3.4**: Given 발행 시트에서 모든 필수 항목이 입력되었을 때, When "발행하기"를 클릭하면, Then 글이 발행되고 해당 임시저장이 자동 제거된다.

### US-4: 단편/시리즈 분류

- **AC-4.1**: Given 발행 시트에서 분류를 확인할 때, When 발행 시트가 열리면, Then "단편"이 기본 선택되어 있다.
- **AC-4.2**: Given 발행 시트에서 시리즈를 선택할 때, When "새 시리즈 만들기"를 클릭하면, Then 시리즈명 입력 필드가 표시되고 시리즈명 입력 후 즉석 생성이 완료된다.
- **AC-4.3**: Given 기존 시리즈를 선택했을 때, When 시리즈가 확정되면, Then 다음 회차 번호가 자동 부여된다.

### US-5: 미리보기

- **AC-5.1**: Given 작가가 미리보기를 클릭할 때, When 미리보기 화면이 로드되면, Then 독자 뷰로 글이 표시된다.
- **AC-5.2**: Given 미리보기 화면에서, When 화면을 확인하면, Then 게시선 유무에 따라 2가지 케이스로 표시된다:
  - **무료 글** (게시선 없음): 게시선 없이 전체 본문이 그대로 표시됨
  - **유료 글** (게시선 있음): 게시선 기준으로 무료 영역은 정상 표시, 유료 영역은 블러 처리 + "구매하면 볼 수 있어요" 안내

### US-6: 임시저장

- **AC-6.1**: Given 에디터에서 글을 작성 중일 때, When 30초가 경과하면, Then 자동 임시저장이 실행되고 하단에 "자동 저장됨 HH:MM"이 표시된다.
- **AC-6.2**: Given 작가가 Cmd/Ctrl+S를 누를 때, When 수동 저장이 실행되면, Then 임시저장 목록에 추가된다. 제한 기준: 새 글(postId=null)은 사용자별 최대 5개, 수정 글(postId 지정)은 해당 글별 최대 5개.
- **AC-6.3**: Given 페이지 이탈(beforeunload) 시, When 이탈 이벤트가 감지되면, Then 현재 내용이 자동 백업된다.
- **AC-6.4**: Given 세션 끊김(heartbeat 실패) 시, When 끊김이 감지되면, Then 마지막 상태가 자동 백업된다.
- **AC-6.5**: Given 임시저장 모달에서 목록을 확인할 때, When 항목을 선택하고 "불러오기"를 클릭하면, Then "현재 내용을 대체합니다" 확인 후 해당 내용이 에디터에 로드된다.
- **AC-6.6**: Given 발행이 완료되면, When 발행 성공 응답을 받으면, Then 해당 글의 임시저장이 목록에서 자동 제거된다.

### US-7: 발행 메타데이터

- **AC-7.1**: Given 발행 시트에서 썸네일 영역을 확인할 때, When 발행 시트가 열리면, Then 자동 생성된 썸네일이 표시되고 "직접 업로드" 옵션이 제공된다.
- **AC-7.2**: Given 발행 시트에서 요약문 영역을 확인할 때, When 발행 시트가 열리면, Then 본문 첫 200자가 자동 추출되어 표시되고 수정이 가능하다.
- **AC-7.3**: Given 발행 시트에서 태그 입력 시, When 태그를 입력하면, Then 태그가 추가되고 삭제할 수 있다.

### US-8: 접근 권한

- **AC-8.1**: Given 발행 시트에서 접근 권한을 확인할 때, When 발행 시트가 열리면, Then "전체공개"가 기본 선택되어 있다.
- **AC-8.2**: Given 접근 권한을 변경할 때, When 전체공개/구독자 전용/구매자 전용/비공개 중 하나를 선택하면, Then 선택한 권한이 발행에 반영된다.

### US-9: 전자책 뷰어 안내 배지

- **AC-9.1**: Given 전자책 뷰어 조건을 충족하는 글(시리즈 + 1000자 이상)일 때, When 발행 시트에서 확인하면, Then "이 글은 전자책 뷰어로 표시됩니다" 안내 배지가 표시된다.
- **AC-9.2**: Given 조건을 충족하지 않는 글일 때, When 발행 시트를 확인하면, Then 안내 배지가 표시되지 않는다.
- **AC-9.3**: Given 안내 배지가 표시되었을 때, When 배지를 확인하면, Then 정보성 표시만 제공되고 클릭/조작은 불가하다.

### US-10: 기존 글 수정

- **AC-10.1**: Given 작가가 기발행된 글의 수정을 선택할 때, When 에디터가 로드되면, Then 기존 내용(제목, 본문, 게시선, 메타데이터)이 모두 복원되고, 현재 발행본의 스냅샷이 자동 생성된다.
- **AC-10.2**: Given 수정 후 업데이트를 클릭할 때, When 업데이트가 완료되면, Then 변경 사항이 즉시 반영되고 해당 포스트의 임시저장(스냅샷 포함)이 자동 정리된다.
- **AC-10.3**: Given 수정 중 작가가 수정 취소를 선택할 때, When 취소가 확인되면, Then 원본 스냅샷(edit_snapshot)이 에디터에 로드되어 변경 이전 상태로 복원된다.

### 콘텐츠 저장 구조: 서버 사이드 Split/Merge

에디터에서는 단일 HTML로 콘텐츠를 작성하고, paywall 마커(`<hr class="paywall-divider">`)가 삽입된 상태로 서버에 전송한다. 서버는 저장 시 마커 기준으로 `freeContent`와 `paidContent`로 분리(split)하여 저장하고, 에디터 로드 시에는 두 필드를 마커와 함께 합쳐서(merge) 단일 HTML로 전달한다. 결제/구독하지 않은 독자에게는 `paidContent`를 전송하지 않아 유료 콘텐츠를 보호한다. 상세 구현은 `implementation-guide.md` 섹션 B.3 참조.

## 9. 엣지 케이스 & 에러 처리

### 에디터

| ID   | 엣지 케이스                              | 처리                                                  |
| ---- | ---------------------------------------- | ----------------------------------------------------- |
| E-01 | 동시 편집 (같은 글을 탭 2개에서 열기)    | Last Write Wins + "다른 탭에서 수정 중" 경고 토스트   |
| E-02 | 대용량 본문 (>100KB)                     | 에디터 성능 경고 + 자동저장 간격 60초로 조정          |
| E-03 | 브라우저 강제 종료 (beforeunload 미실행) | 마지막 자동 저장본(최대 30초 전)으로 복구             |
| E-04 | CKEditor 로드 실패                       | "에디터를 불러올 수 없습니다" + 재로드 버튼           |
| E-05 | 게시선 여러 개 삽입 시도                 | 게시선은 1개만 허용, 기존 게시선 있으면 버튼 비활성화 |

### 임시저장

| ID   | 엣지 케이스                       | 처리                                                                                                  |
| ---- | --------------------------------- | ----------------------------------------------------------------------------------------------------- |
| E-06 | 임시저장 5개 초과                 | 새 글/수정 글 각각 5개 기준으로 가장 오래된 자동 저장본 삭제 (수동 저장본 및 edit_snapshot 우선 보존) |
| E-07 | 자동저장 중 네트워크 끊김         | localStorage 백업 + 네트워크 복구 후 서버 동기화                                                      |
| E-08 | 임시저장 불러오기 시 현재 작성 중 | "현재 내용을 대체합니다. 계속하시겠습니까?" 확인 팝업                                                 |
| E-09 | 빈 내용 자동저장                  | 제목+본문 모두 비어있으면 자동저장 스킵                                                               |

### 발행

| ID   | 엣지 케이스                        | 처리                                                          |
| ---- | ---------------------------------- | ------------------------------------------------------------- |
| E-10 | 유료 -> 무료 전환 시 기구매자 존재 | "nn명이 이미 구매했습니다" 경고 팝업, 작가 재량으로 전환 허용 |
| E-11 | 시리즈 회차 번호 충돌              | 서버 측 자동 증분 (트랜잭션 내 처리로 race condition 방지)    |
| E-12 | 필수 항목 미입력 (제목 없음)       | 발행 버튼 비활성화 + "제목을 입력해주세요" 에러 메시지        |
| E-13 | 발행 중 네트워크 에러              | 재시도 버튼 + 임시저장은 유지                                 |
| E-14 | 썸네일 업로드 실패                 | 기본 썸네일(자동 생성)로 발행 가능 + 재업로드 안내            |
| E-15 | 즉석 시리즈명 중복                 | 같은 서점 내 동일 시리즈명 허용 (seriesId로 구분)             |

### 보안

| ID   | 엣지 케이스                      | 처리                                                       |
| ---- | -------------------------------- | ---------------------------------------------------------- |
| E-16 | 타인 글 수정 시도                | post.bookstoreId -> bookstore.userId === ctx.user.sub 검증 |
| E-17 | XSS (본문에 스크립트 삽입)       | CKEditor 내장 sanitizer + 서버 측 HTML 정화 (DOMPurify)    |
| E-18 | 대용량 이미지 업로드             | 파일 크기 제한 (5MB per image) + 서버 측 리사이즈          |
| E-19 | 서점 미보유 유저가 API 직접 호출 | Service 레이어에서 bookstore 소유 여부 검증                |

## 10. 우선순위 & 스코프

### MVP (이번 Feature)

글쓰기 에디터 + 유료 게시선 + 발행 플로우 + 임시저장 + 미리보기

| 항목       | 범위                                                 |
| ---------- | ---------------------------------------------------- |
| Entity     | DraftEntity 신규, PostEntity 확장, SeriesEntity 신규 |
| 에디터     | CKEditor 5 WYSIWYG, 기본 서식, 이미지 삽입           |
| 게시선     | 삽입/삭제/드래그, 의도 추론 (삽입=유료, 삭제=무료)   |
| 발행       | 2단계 발행 시트, Smart Default (무료/전체공개/단편)  |
| 시리즈     | 단편/시리즈 분류, 즉석 시리즈 생성, 회차 자동 부여   |
| 접근 권한  | 전체공개/구독자/구매자/비공개                        |
| 임시저장   | 30초 자동 + 이탈/세션 백업 + 수동 5개 + 모달 CRUD    |
| 메타데이터 | 썸네일(자동/업로드), 요약문(자동 추출), 태그         |
| 미리보기   | 독자 뷰, 유료 영역 구분                              |
| 안내 배지  | 전자책 뷰어 판정 결과 표시                           |

### Growth (다음 이터레이션)

| 항목               | 범위                           |
| ------------------ | ------------------------------ |
| 예약 발행          | 시간 지정 자동 공개            |
| AI 글쓰기 지원     | 맞춤법 교정, 문체 제안         |
| 이미지 고급 편집   | 크롭, 리사이즈, 캡션           |
| 발행 후 수정 이력  | 변경 내역 추적                 |
| 시리즈 회차 재배치 | 회차 순서 변경                 |
| 마크다운 임포트    | .md 파일 업로드 -> 에디터 변환 |
| AI 요약문/태그     | 자동 생성                      |
