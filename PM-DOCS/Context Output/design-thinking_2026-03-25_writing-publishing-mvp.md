# Design Thinking: 글쓰기-발행 에디터 MVP

> 생성일: 2026-03-25
> 워크플로우: design-thinking
> 디자인 챌린지: 어떻게 하면 작가가 글쓰기에만 몰입하면서도, 유료/무료 발행과 시리즈 관리를 자연스럽게 해낼 수 있을까?
> 대상 사용자: Readly 에디터 (GL 중견 아마추어 작가)
> 관련 Feature: F3 글쓰기-발행 에디터

---

## 요약

### 핵심 인사이트

1. **유료/무료 결정이 가장 큰 심리적 부담** -- 작가는 "돈 받는 글"을 선언하는 순간 긴장한다. 게시선(paywall line)으로 이 결정을 "글쓰기 행위의 일부"로 녹이면 부담이 사라진다
2. **임시저장 유실 = 작가에게 치명적 트라우마** -- 브라우저 꺼짐, 세션 만료로 글을 잃은 경험이 플랫폼 이탈의 직접 원인. 30초 자동저장 + 이탈 백업이 "보이지 않는 안전망"으로 필수
3. **발행 과정의 복잡성이 글 생산량을 좌우** -- 메타데이터, 분류, 가격 설정이 번거로우면 글 자체를 덜 쓰게 됨. Smart Default로 "확인만 하면 발행"되는 경험 필요
4. **시리즈 관리 허들이 단편 편향을 만든다** -- 별도 시리즈 관리 페이지에서 미리 생성해야 하면, 작가는 일단 단편으로 발행. 발행 시점에 즉석 시리즈 생성이 가능해야 시리즈 전환율 상승
5. **전자책 뷰어는 에디터의 관심사가 아니다** -- 작가는 "내 글이 어떻게 보이는지"만 궁금할 뿐, 뷰어 자체를 조작하려 하지 않음. 안내 배지 수준이면 충분

## POV Statement

**작가 POV**: "작가는 글쓰기에 몰입한 채로 유료/무료 결정과 발행 설정을 자연스럽게 마칠 수 있는 에디터가 필요하다. 왜냐하면 현재의 발행 과정은 글쓰기 흐름을 끊고, 임시저장 유실 불안과 복잡한 설정 피로가 글 생산량을 직접적으로 줄이기 때문이다."

## 솔루션 컨셉

### 컨셉 1: "투명한 에디터" (Transparent Editor)

글쓰기 몰입을 방해하는 모든 요소를 제거하거나 자동화한다.

- CKEditor 5 기반 WYSIWYG -- 미니멀 툴바, 필수 서식만 노출
- 유료 게시선 = 에디터 내 시각적 구분선 -- 삽입/삭제/드래그
- 게시선 유무로 유료/무료 자동 판정 (의도 추론)
- 에디터 상단 상태 바: "유료 발행 예정" / "전자책 뷰어 표시" 안내
- 하단 저장 상태: "자동 저장됨 14:32" 조용한 표시

### 컨셉 2: "2단계 발행 시트" (Two-Step Publishing)

글쓰기(에디터)와 발행(설정)을 명확히 분리하되, 발행을 최소 클릭으로 완료한다.

- **Step 1**: 에디터에서 글쓰기에만 집중
- **Step 2**: "발행하기" 클릭 시 발행 시트(바텀시트) 등장
  - 분류: 단편(기본) / 시리즈 선택 / 즉석 시리즈 생성
  - 가격: 게시선 있으면 자동 유료 (기본 300원) / 없으면 무료
  - 접근 권한: 전체공개(기본) / 구독자 / 구매자 / 비공개
  - 메타데이터: 썸네일(자동 or 업로드), 요약문(자동 추출), 태그
  - 전자책 뷰어 안내 배지 (조건 충족 시)
  - 미리보기 버튼 -> 독자 뷰 시뮬레이션

### 컨셉 3: "보이지 않는 안전망" (Invisible Safety Net)

모든 단계에서 글 유실을 방지하는 다층 안전 시스템.

- **Layer 1 -- 자동 임시저장**: 30초 주기, debounce (타이핑 중이면 완료 후)
- **Layer 2 -- 이탈 감지**: beforeunload + visibilitychange 이벤트 -> 즉시 저장
- **Layer 3 -- 세션 백업**: heartbeat 실패 감지 -> 마지막 상태 자동 백업
- **Layer 4 -- 수동 임시저장**: Cmd/Ctrl+S, 최근 5개 버전 관리
- **Layer 5 -- 임시저장 모달**: 목록 조회 / 불러오기 / 삭제 / 마지막 저장 시간
- **정리**: 발행 완료 시 해당 글의 임시저장 목록 자동 제거

```

    |
    v

    |                          |
    v                          v
[빈 에디터] <------------------+
    |

    |
    v

    |
    v

    |--- 분류: 단편(기본) / 시리즈 선택 / [즉석 시리즈 생성]
    |--- 가격: 자동 설정 (게시선 유무 기반)
    |--- 접근 권한: 전체공개(기본)
    |--- 메타데이터: 썸네일/요약문/태그 (Smart Default)
    |--- 전자책 뷰어 안내 배지 (조건부)
    |
    |--- [미리보기] --> [미리보기 화면] (독자 뷰 / 작가 뷰)
    |                       |
    |                  [에디터로 돌아가기]
    |
    v

    |
    v

```

| ----------------------------------------- | ------------ | ----------------------------------------- |

| 30초 자동저장이 성능에 영향을 주지 않는다 | High         | 기술 프로토타입 벤치마크 (대용량 글 기준) |
| 2단계 발행 시트가 번거롭지 않다           | High         | Task 완료 시간 측정 + Think-aloud         |
| Smart Default가 작가 의도와 일치한다      | Medium       | 발행 후 설정 수정 빈도 추적               |
| 즉석 시리즈 생성이 실제로 활용된다        | Medium       | 사용 빈도 추적 (시리즈 생성 경로 분석)    |

## 테스트 계획

### MVP 완료 기준 (Definition of Done)

**에디터 Core**:

- [ ] CKEditor 기반 리치 텍스트 에디터가 동작한다
- [ ] 제목 + 본문 입력이 가능하다
- [ ] 기본 서식 (B/I/U, 제목, 링크, 이미지 삽입)이 동작한다
- [ ] 유료 게시선을 삽입/삭제/드래그 이동할 수 있다
- [ ] 게시선 삽입 시 자동 유료, 삭제 시 자동 무료로 전환된다

**임시저장 시스템**:

- [ ] 30초마다 자동 임시저장이 실행된다
- [ ] 페이지 이탈(beforeunload) 시 자동 저장된다
- [ ] 세션 끊김 시 마지막 상태가 백업된다
- [ ] 수동 임시저장(Cmd/Ctrl+S)이 동작한다
- [ ] 임시저장 목록(최근 5개)을 모달에서 조회할 수 있다
- [ ] 임시저장 불러오기/삭제가 가능하다
- [ ] 발행 완료 시 해당 임시저장이 목록에서 제거된다

**발행 플로우**:

- [ ] 발행 시트에서 단편/시리즈 분류를 선택할 수 있다
- [ ] 즉석 시리즈 생성이 동작한다
- [ ] 시리즈 선택 시 회차 번호가 자동 부여된다
- [ ] 가격 설정이 가능하다 (게시선 기반 기본값 자동)
- [ ] 접근 권한을 선택할 수 있다
- [ ] 썸네일 자동생성 또는 직접 업로드가 가능하다
- [ ] 요약문이 본문 첫 200자에서 자동 추출된다
- [ ] 태그를 입력할 수 있다
- [ ] 전자책 뷰어 안내 배지가 조건 충족 시 표시된다

**미리보기**:

- [ ] 독자 뷰로 글 미리보기가 가능하다
- [ ] 유료 게시선 기준으로 무료/유료 영역이 구분 표시된다

### 검증 시나리오

**시나리오 1: 첫 글 발행 (단편, 무료)**

- 작가가 에디터를 열고 제목과 본문을 작성한다
- 게시선을 삽입하지 않는다
- 발행하기 -> 발행 시트에서 기본값(무료, 전체공개, 단편) 확인
- 메타데이터(요약문 자동, 태그 입력) 설정 -> 발행
- **기대**: 5분 이내 첫 글 발행 완료

**시나리오 2: 유료 시리즈 글 발행**

- 작가가 글을 쓰고 중간에 유료 게시선을 삽입한다
- 에디터 상태 바에 "유료 발행" 표시 확인
- 발행 시트에서 시리즈 선택 -> "새 시리즈 만들기" -> 생성
- 회차 #1 자동 부여, 가격 유료 자동, 썸네일 설정
- 미리보기에서 무료/유료 영역 구분 확인 -> 발행
- **기대**: 첫 시리즈 생성 + 유료 발행까지 10분 이내

**시나리오 3: 임시저장 복구**

- 작가가 글쓰기 중 브라우저를 실수로 닫는다
- 다시 에디터에 진입하면 "이전 임시저장이 있습니다" 안내
- 임시저장 모달에서 가장 최근 자동 백업 확인 -> 불러오기
- **기대**: 마지막 타이핑 후 최대 30초 이내 내용까지 복구

**시나리오 4: 임시저장 관리**

- 작가가 여러 글을 번갈아 작업한다
- 임시저장 모달에서 5개 목록 확인
- 원하는 글 불러오기, 불필요한 글 삭제
- **기대**: 목록에서 즉시 식별 가능 (제목, 시간, 글자 수)

**시나리오 5: 게시선 의도 추론**

- 게시선 삽입 -> 유료 자동 전환 확인
- 게시선 삭제 -> 무료 자동 복귀 확인
- 게시선 위치 드래그 변경 -> 미리보기에서 변경 반영 확인
- **기대**: 유료/무료 전환이 게시선 유무와 100% 동기화

**시나리오 6: 전자책 뷰어 안내 배지**

- 1000자 이상 시리즈 글 작성 -> "전자책 뷰어로 표시됩니다" 배지 확인
- 500자 단편 -> 배지 미표시 확인
- **기대**: 배지가 정보성으로만 표시, 클릭 불가

### 피드백 프레임워크 (관찰 포인트 3가지)

**1. 행동 (What they DO)**

- 게시선 삽입 시 머뭇거리는가? (직관성)
- 발행 시트에서 어디를 가장 먼저 보는가? (정보 위계)
- 임시저장 모달을 스스로 찾는가? (발견성)

**2. 발화 (What they SAY)**

- "이건 뭐지?" 순간 (혼란 포인트)
- "오, 이건 좋다" 순간 (가치 포인트)
- "이건 없나?" 순간 (결핍 포인트)

**3. 감정 (What they FEEL)**

- 글쓰기 몰입도 -- 방해 요소가 있는가
- 발행 완료 시 만족감 -- 달성감 or 안도감 or 불안감
- 임시저장 복구 시 안도감 수준

### Pivot 시나리오

| 테스트 결과              | Pivot 방향                                                 |
| ------------------------ | ---------------------------------------------------------- |
| 게시선이 직관적이지 않음 | 게시선 대신 "유료 설정" 토글 전환, 미리보기에서 영역 표시  |
| 2단계 발행이 번거로움    | 에디터 사이드바에 발행 설정 통합 (1단계 발행)              |
| 자동저장 30초가 불안함   | 10초로 단축 또는 변경 감지 즉시 저장 (debounce 3초)        |
| 즉석 시리즈가 잘 안 쓰임 | 발행 시트에서 제거, 별도 시리즈 관리 페이지로 이동         |
| 메타데이터가 부담됨      | 메타데이터 전체를 선택사항으로, 기본값만으로 발행 가능하게 |
