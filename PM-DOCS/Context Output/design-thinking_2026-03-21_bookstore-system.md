# Design Thinking: 서점 시스템

> 생성일: 2026-03-21
> 워크플로우: design-thinking
> 디자인 챌린지: 다른 플랫폼/포맷의 작가가 Readly 서점을 손쉽게 오픈·운영하고, 작품 큐레이션·시리즈 세팅을 자유롭게 할 수 있게 만들기
> 대상 사용자: GL 중견 아마추어 작가 + GL 독자
> 관련 Feature: F2 서점 시스템

## 요약

### 핵심 인사이트

1. **작가-독자 인식 갭**: 작가는 "독자가 이중결제하면 미안"하지만, GL 독자는 실제로 유료 결제에 거리낌이 없음
2. **블로그가 아닌 도서관**: 기존 아마추어 플랫폼(포스타입, 네이버 블로그)의 블로그 형태에서 탈피, Wattpad 같은 썸네일 기반 도서관 UI를 원함
3. **GL 장르의 희소성**: GL 콘텐츠 자체가 부족한 시장이므로, 독자는 GL이 있는 곳이면 어디든 가는 "하이에나" 패턴
4. **Cold Start 해법**: 작가 스카우트 → GL 콘텐츠 볼륨 확보 → 독자 자연 유입 순서
5. **발행 디폴트 세팅**: 30편+ 작품을 관리하는 작가에게 단편/시리즈별 기본 발행 설정은 필수

## POV Statement

**작가 POV**: "다른 플랫폼 GL 작가는 흩어진 작품을 한 곳에 모아 내만의 서점을 운영하고 싶다. 왜냐하면 여러 플랫폼에서의 수고와 비싼 수수료에 지쳤고, 내 작품을 내가 원하는 대로 보여주고 싶기 때문이다."

**독자 POV**: "GL 독자는 퀄리티 높은 GL 콘텐츠가 모여있는 플랫폼이 필요하다. 왜냐하면 없기 때문이다."

## 솔루션 컨셉

### 1. 서점 UI (도서관 스타일)

```
[서점 메인]
├── 장르 태그 배지 (서점 정체성 - 즉시 인지)
├── 최신 발행 섹션 (썸네일 그리드)
├── 인기 글 섹션 (썸네일 그리드)
└── 탐색 메뉴
    ├── 단편 전체보기
    └── 시리즈 전체보기
```

### 2. 독자 반응 시스템

| #   | 기능                       | 위치        | 설명                                                                                                  |
| --- | -------------------------- | ----------- | ----------------------------------------------------------------------------------------------------- |
| 1   | 서점 리뷰                  | 서점 페이지 | 서점 전체에 대한 응원/코멘트                                                                          |
| 2   | 편당 별점                  | 작품 상세   | 각 편마다 별점                                                                                        |
| 3   | 편당 댓글                  | 작품 상세   | 각 편마다 댓글                                                                                        |
| 4   | 스크랩 표기                | 독자 서재   | 독자가 스크랩하면 표시                                                                                |
| 5   | **문장 하이라이트 코멘트** | 전자책 뷰어 | 최대 2문장 하이라이트 → 문장 댓글 → 편당 댓글로 집계 → 독자 서재에서도 확인 → 다른 독자에겐 요약 표시 |

### 3. 발행 디폴트 세팅

- **단편 기본값**: 유료/무료, 구매 가격, 성인/전체 이용가
- **시리즈 기본값**: 시리즈 생성 시 설정, 회차 발행 시 자동 적용
- 매 편 발행 시 디폴트에서 개별 수정 가능

### 4. 마이그레이션

- **MVP**: Word/HWP 파일 업로드 + 수동 복사-붙여넣기
- **Growth**: 브라우저 확장 프로그램 (포스타입 + 투비컨티뉴드)
  - 작가가 로그인 상태에서 버튼 하나로 내 글 전부 추출
  - 구조화된 파일(JSON/ZIP)로 패키징 → Readly에 업로드

## 프로토타입

### 핵심 경험 흐름

```
1. 서점 오픈: 필명 + 서점명 입력 → 약관 동의 → 서점 생성
2. 프로필 설정: 소개글 + 프로필 이미지
3. 발행 디폴트 설정:
   ├── 단편 기본값: 유료/무료, 가격, 성인/전체
   └── (시리즈별 기본값은 시리즈 생성 시 설정)
4. 작품 업로드: Word/HWP 업로드 또는 직접 입력
5. 시리즈 세팅: 시리즈 생성 + 디폴트 설정(유료/가격/등급) + 썸네일
6. 발행: 디폴트 적용 → 편별 수정 가능 → 공개!
```

### 핵심 가정

| 가정                                             | 위험도       |
| ------------------------------------------------ | ------------ |
| GL 작가들이 실제로 Readly로 옮기고 싶어할 것이다 | **Critical** |
| 독자들이 새 플랫폼에서도 결제할 것이다           | High         |
| Word/HWP 업로드만으로 MVP에 충분할 것이다        | Medium       |
| 10% 수수료가 경쟁력 있을 것이다                  | Medium       |
