## 요약

1. **MVP 핵심은 "글로벌 작품의 열람과 판매 경계를 허무는 것"** -- 실시간 번역, 연재 열람, 유료 판매, 로컬라이제이션의 4대 축이 최소 기능 세트
2. **현재 구현된 기초 위에 쌓아야 할 Gap이 명확함** -- 인증/Post CRUD/Follow/접근권한/에디터UI는 존재하나, 번역/결제/구독/프로필/탐색이 미구현
3. **팬픽션 니치 시장의 특수성(커플링 탐색, 시한부 콘텐츠, GL 갈증)이 기능 우선순위를 결정**해야 함
4. **Cold Start를 돌파하려면 작가 온보딩 경험과 독자 탐색 경험이 동시에 MVP에 포함**되어야 양면 시장이 작동
5. **"태국에서 340명이 당신의 글을 읽었습니다"라는 한 줄이 작가 Lock-in의 전부** -- 글로벌 리치 피드백이 MVP에서부터 존재해야 함

## 세션 셋업

### 주제 및 목표

- **주제**: Readly MVP에 포함할 기능의 발산 및 우선순위 결정
- **목표**: Product Brief의 Phase 1(MVP) 로드맵을 구체적 기능 단위로 분해하여, PRD 작성의 입력으로 활용
- **제약**: 3개월 개발 기간, 소규모 팀 (개발자 1-2명 가정), 기존 구현 기반 위에 빌드

### 현재 구현 상태 진단

| 영역         | 구현 상태  | 세부                                                               |
| ------------ | ---------- | ------------------------------------------------------------------ |
| 인증         | 구현 완료  | JWT (Access 15분, Refresh 7일), 이메일/비밀번호                    |
| Post CRUD    | 구현 완료  | 생성/수정/발행/취소/삭제, freeContent/paidContent 분리             |
| 접근권한     | 구현 완료  | public/subscriber/purchaser/private 4단계                          |
| 에디터 UI    | 구현 완료  | CKEditor 5, 포스트 목록 관리 (필터/검색/액션)                      |
| Follow       | 구현 완료  | 팔로우/언팔로우, 카운트 조회                                       |
| SEO          | 구현 완료  | 빌드 타임 프리렌더링 (Vite Plugin + Puppeteer + Helmet)            |
| 결제         | **미구현** | 정책만 정의 (포스트 구매, 월간 구독)                               |
| 구독         | **미구현** | subscriber 접근권한 정의만 존재                                    |
| 번역         | **미구현** | Product Brief에서 핵심 차별화로 정의                               |
| 프로필       | **부분**   | User Entity에 nickname/profileImage 존재, 프로필 페이지 미확인     |
| 탐색/검색    | **미구현** | getAccessiblePosts()가 public+published만 반환                     |
| 독자 열람 UI | **부분**   | freeContent 표시 + paidContent 잠금 로직 존재, 독자 전용 뷰 미확인 |

## 기법 실행

### 기법 1: SCAMPER 분석

Readly의 핵심 가치 체인(작가가 쓴다 -> 번역된다 -> 독자가 찾는다 -> 결제한다 -> 읽는다)에 SCAMPER를 적용한다.

#### S - Substitute (대체)

| #   | 아이디어            | 설명                                                                        |
| --- | ------------------- | --------------------------------------------------------------------------- |
| 1   | **AI 번역 대체 UI** | 전문 번역 대신 AI 실시간 번역을 기본으로, 원문+번역 토글 제공               |
| 2   | **크레딧 시스템**   | 직접 현금 결제 대신 플랫폼 크레딧(코인)으로 통일하여 결제 마찰 감소         |
| 3   | **소셜 로그인**     | 이메일 가입 대신 Google/Twitter/카카오톡 소셜 로그인으로 가입 허들 제거     |
| 4   | **마크다운 에디터** | WYSIWYG 대신 마크다운 기반 작성 (작가 타겟이 글쓰기 중심이므로 가벼운 도구) |

| 5   | **이중 리더(Dual Reader)**    | 원문과 번역을 나란히 보여주는 읽기 모드 (학습 + 감상 결합)               |

### 우선순위 평가 (Impact x Feasibility)

#### 평가 기준

- **Impact** (1-5): 사용자 가치 + 비즈니스 가치 + 차별화 기여도
- **Feasibility** (1-5): 개발 복잡도 역수 + 기존 구현 활용도 + 3개월 내 실현 가능성
- **Score** = Impact x Feasibility

## 상위 선정: MVP Feature 구성안

### Tier 1: Must-Have (MVP Core) -- 이것 없이는 MVP가 아님

| Feature                            | 설명                                                                                                                | 근거                                                                        |
| ---------------------------------- | ------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------- |
| **F1. 시리즈/연재 시스템**         | Post를 시리즈(Series)로 묶는 구조. 시리즈 > 챕터(Post) 계층. 시리즈별 표지/설명/태그.                               | 팬픽은 연재물이 기본 단위. 단독 Post만으로는 핵심 사용 사례를 지원하지 못함 |
| **F2. AI 실시간 번역 + 이중 리더** | Post 열람 시 AI 번역 자동 제공. 원문/번역 나란히 보기(Side-by-side) 또는 토글. MVP 지원 언어: 한국어, 영어, 태국어. | Product Brief의 핵심 차별화. "Write Once, Reach Global"의 기술적 실현       |
| **F3. 태그 + 커플링 탐색**         | 자유 태그 + 팬픽 특화 커플링 태그. 태그 기반 필터링/탐색. 인기 태그 표시.                                           | 팬픽 독자의 핵심 탐색 방식. 커플링 없는 팬픽 플랫폼은 존재 의의가 없음      |
| **F4. 크레딧 결제 시스템**         | 플랫폼 크레딧(코인) 충전 -> 포스트 구매/구독에 사용. 충전: 토스페이먼츠 연동. 구독: 월간 자동 갱신.                 | 수익화의 기본 인프라. 결제 없이는 비즈니스 모델이 작동하지 않음             |
| **F5. 큐레이션 메인 피드**         | 메인 페이지에 인기/최신/추천 탭. 태그 필터 결합. 비로그인 사용자도 열람 가능.                                       | 독자가 좋은 글을 "발견"하는 첫 번째 접점. Cold Start 시 콘텐츠 노출이 핵심  |
