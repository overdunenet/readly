# 사용자 팔로우

## 목적

팔로워가 관심 있는 에디터를 팔로우하여 지속적인 콘텐츠 관계를 형성합니다. 팔로우는 Readly 플랫폼에서 에디터-팔로워 연결의 기본 단위이며, 향후 구독/결제 기능의 기반이 됩니다.

## 팔로우 대상: 유저 (멀티 서점 대비)

팔로우 대상은 **서점(Bookstore)이 아닌 유저(User)**입니다. FollowEntity의 followerId/followeeId는 모두 UserEntity.id를 참조합니다. 향후 1인 멀티 서점을 지원하더라도 팔로우 관계는 유저 단위로 유지되어, 특정 에디터의 모든 서점 활동을 하나의 팔로우로 추적할 수 있습니다.

## 핵심 기능

| 기능                  | 설명                              | 사용자 관점                                          |
| --------------------- | --------------------------------- | ---------------------------------------------------- |
| 팔로우                | 에디터를 팔로우하여 관계 형성     | 에디터 프로필에서 "팔로우" 버튼 클릭                 |
| 언팔로우              | 팔로우 관계 해제                  | "팔로잉" 버튼 호버 시 "언팔로우" 표시, 클릭으로 해제 |
| 팔로워/팔로잉 수 조회 | 에디터의 인기도 및 활동 범위 확인 | 프로필 페이지에서 카운트 표시 (로그인 불필요)        |
| 팔로우 상태 확인      | 현재 팔로우 여부 표시             | 버튼이 "팔로우" 또는 "팔로잉"으로 자동 전환          |

## 사용자 흐름

1. 팔로워가 에디터 프로필 방문 → 팔로우 버튼 표시 (로그인 상태에서만)
2. "팔로우" 버튼 클릭 → 즉시 "팔로잉"으로 변경 (빠른 피드백)
3. "팔로잉" 버튼에 마우스 호버 → "언팔로우"로 변경 (빨간색)
4. "언팔로우" 클릭 → 팔로우 해제, 버튼이 "팔로우"로 복원
5. 자기 자신의 프로필에서는 팔로우 버튼 미표시

## 비즈니스 규칙

- 로그인한 사용자만 팔로우/언팔로우 가능
- 자기 자신은 팔로우 불가
- 동일 에디터를 중복 팔로우 불가
- 팔로워/팔로잉 카운트는 비로그인 사용자도 조회 가능
- 사용자 계정 삭제 시 관련 팔로우 관계 자동 정리
