# 배포 효율화

## 목적

코드 변경이 프로덕션 환경에 반영되는 시간을 단축하여, 에디터와 팔로워에게 새로운 기능과 수정 사항을 빠르게 제공한다.

## 핵심 기능

| 기능 | 설명 | 사용자 관점 |
|------|------|------------|
| 의존성 캐싱 | yarn 패키지 캐시를 CI 실행 간 재사용 | 빌드 시간 단축으로 빠른 기능 배포 |
| Docker 레이어 캐시 | 변경되지 않은 Docker 레이어를 재사용 | 동일 의존성일 때 이미지 빌드 시간 대폭 감소 |
| 직접 이미지 Push | 빌드와 동시에 레지스트리에 직접 전송 | 로컬 저장 + 별도 push 대비 시간 절약 |

## 사용자 흐름

1. 개발자가 main 브랜치에 코드를 push → CI가 캐시된 의존성으로 빠르게 빌드
2. Docker 이미지가 변경된 레이어만 재빌드하여 ECR에 직접 push
3. Elastic Beanstalk가 새 이미지로 환경 업데이트 → 사용자에게 변경 반영
